완료된 작업을 `docs/ROADMAP.md`에 반영한다.

**입력**: $ARGUMENTS
(예: `Task 007` 또는 `Task 007, Task 008` 또는 `Phase 3`)

## 실행 절차

### 1. ROADMAP.md 읽기

`docs/ROADMAP.md` 전체를 읽어 현재 상태를 파악한다.

### 2. 완료 표시 업데이트

`$ARGUMENTS`에 명시된 태스크(또는 Phase)를 찾아 아래 규칙으로 수정한다:

**태스크 라인 패턴:**
```
변경 전: - **Task 007: Notion API 통합 구현**
변경 후: - **Task 007: Notion API 통합 구현** ✅ - 완료
```

**하위 항목:**
```
변경 전:   - Notion API 클라이언트 설정
변경 후:   - ✅ Notion API 클라이언트 설정
```

**Phase 헤더 (해당 Phase의 모든 Task가 완료된 경우에만):**
```
변경 전: ### Phase 3: 핵심 기능 구현
변경 후: ### Phase 3: 핵심 기능 구현 ✅
```

### 3. 헤더 상태 업데이트

파일 5번째 줄의 `> **마지막 업데이트**` 라인을 오늘 날짜와 현재 상태로 갱신한다:

```
> **마지막 업데이트**: YYYY-MM-DD | **현재 상태**: [완료 Phase] 완료, [다음 Phase] 진행 예정
```

오늘 날짜는 `YYYY-MM-DD` 형식을 사용한다.

### 4. 변경 이력 추가

파일 하단 `## 변경 이력` 테이블에 새 행을 마지막 행 다음에 추가한다:

```markdown
| YYYY-MM-DD | [완료 Task/Phase 요약]. [다음 단계] 진행 예정. |
```

### 5. MVP 체크리스트 업데이트 (해당 시)

완료된 태스크와 연관된 F-코드 항목을 `[ ]` → `[x]` 로 변경한다.

연관 매핑:
- Task 007, 008 완료 → F001, F002 체크
- Task 009 완료 → F003 체크
- Task 005 완료 → F012 체크

### 6. 결과 요약 출력

변경된 항목을 간략히 나열한다:
- 완료 표시된 태스크 목록
- 업데이트된 헤더 상태
- 추가된 변경 이력
